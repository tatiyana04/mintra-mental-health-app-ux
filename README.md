# Mintra — Mental Health Support App (UX/UI Case Study)

**A stigma-free mental health platform for youth, built with user-centered design.**

[View the Figma Prototype](https://www.figma.com/design/KUxNARjLr0NHZwsDipYwLb/Mintra?node-id=1-13298)

## Overview

Mintra is a mobile app concept designed to help young people in Sri Lanka access mental health support without the stigma, cost, and long wait times that block them from professional care. It connects users to therapists, peer support groups, mood/habit tracking, educational content, and crisis intervention tools — all in one place.

This was a group project (100+ screens) built over two phases: a full UX research + design phase, followed by a usability testing phase where we ran moderated tests, A/B tested key screens, and iterated the high-fidelity prototype based on real user feedback.

## The Problem

Youth mental health is heavily under-addressed in Sri Lanka. Existing solutions (BetterHelp, Headspace) are too expensive, too clinical, or don't account for cultural stigma. Through interviews and a 105-response survey, we found:

- 78% of respondents reported mental health struggles (anxiety, depression, stress)
- Most users preferred instant, one-tap access to crisis support over navigating menus
- Users wanted anonymity, gamified progress tracking, and culturally relevant content

## Process

We followed a Design Thinking approach: **Empathize → Define → Ideate → Prototype → Test → Implement**

- **Research:** User interviews, a 105-response survey, competitor analysis (BetterHelp, Clear Fear, Move Mood, MindShift)
- **Define:** 5 personas, user journeys, empathy maps, pain-point mapping, problem statements
- **Ideate:** Affinity mapping, "How Might We" statements, feature prioritization
- **Prototype:** Low-fidelity wireframes → high-fidelity interactive prototype in Figma
- **Test:** Expert heuristic reviews, then structured usability testing (9 participants) with A/B testing on key flows
- **Iterate:** Refined the crisis support (SOS) flow, journaling screen, and therapist booking flow based on test results

## Key Features

- Mood tracking with journaling and a dynamic emotion slider
- Therapist discovery and appointment booking
- One-tap crisis support with a slide-to-confirm SOS button (redesigned after usability testing revealed accidental triggers)
- Peer support groups / social circles
- Gamified 30-day challenges and streaks
- Anonymous mode and guardian-approval flow for minors

## Usability Testing Highlights

We ran moderated usability tests and A/B tested several screens against the original design. Notable improvements:

- **Crisis Support:** Switched from a single-tap SOS button to a slide-to-send gesture — reduced accidental triggers and increased user trust in high-stress scenarios
- **Journaling:** Separated the entry field from historical affirmations to reduce cognitive load
- **Therapist Booking:** Split "View Profile" and "Book Now" into distinct steps, improving task completion clarity
